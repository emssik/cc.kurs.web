# Lekcja 02.10 — MCP Part 2: Instalacja i Pierwsze Kroki (Research dla podcastu)

## Trudne tematy techniczne

- **Konfiguracja JSON (pliki .claude.json, settings.json, settings.local.json)** — format danych z nawiasami klamrowymi, cudzysłowami i zagnieżdżeniami. Osoby nietechniczne mogą nie wiedzieć czym jest JSON ani jak go edytować, a drobna literówka (brak przecinka, nadmiarowy nawias) powoduje niedziałającą konfigurację.

- **npx, npm, Node.js i ich wzajemne zależności** — lekcja zakłada uruchamianie serwerów przez `npx`, ale osoba nietechniczna może nie wiedzieć czym jest Node.js, dlaczego trzeba go instalować, co to jest "pakiet npm" ani czym różni się `npx` od `npm`.

- **Transport: stdio vs HTTP** — koncepcja "standardowego wejścia/wyjścia" i komunikacji między procesami przez "rurki" (pipes) jest abstrakcyjna. Różnica między lokalnym a zdalnym połączeniem wymaga zrozumienia jak procesy komunikują się w systemie operacyjnym.

- **Zmienne środowiskowe (environment variables) i ich ustawianie** — pojęcia takie jak `export GITHUB_TOKEN=...`, `${GITHUB_TOKEN}` jako placeholder, pliki `.env`, narzędzia typu `direnv` — to warstwa systemu operacyjnego, z którą osoby nietechniczne rzadko mają do czynienia.

- **Tokeny API, OAuth i scopes (zakresy uprawnień)** — różnica między tokenem a hasłem, przepływ OAuth (Google Cloud Console, Client ID, Desktop app), koncepcja "zakresów uprawnień" (np. `repo`, `channels:history`) — to zagadnienia z pogranicza bezpieczeństwa i architektury aplikacji.

- **Ścieżki absolutne vs względne, tilde (~), backslash na Windows** — lekcja wielokrotnie podkreśla problemy ze ścieżkami: `~` nie działa w JSON, trzeba używać pełnych ścieżek, na Windows podwójny backslash. Osoby nietechniczne mogą nie rozumieć systemu plików na tym poziomie.

- **PATH w systemie operacyjnym** — lekcja mówi "npx nie jest w PATH" jako przyczynę błędu. Koncept zmiennej PATH i tego jak system wie gdzie szukać programów jest nieoczywisty.

- **Sandbox i jego granice** — różnica między uruchamianiem komend Bash "w sandboxie" a serwerów MCP "poza sandboxem" wymaga zrozumienia izolacji procesów, uprawnień systemowych i modelu zagrożeń.

- **Prompt Injection przez dane** — atak polegający na ukrywaniu instrukcji w danych (np. README), które Claude interpretuje jako polecenia użytkownika. Wymaga zrozumienia jak działa model językowy i dlaczego nie odróżnia "danych" od "poleceń".

- **Git, commity, .gitignore i historia repozytorium** — lekcja ostrzega przed commitowaniem sekretów, ale osoby nietechniczne mogą nie rozumieć czym jest commit, dlaczego usunięcie pliku nie usuwa go z historii gita, ani jak działa .gitignore.

- **SQLite i zapytania SQL** — choć lekcja mówi "nie musisz znać SQL", sam koncept lokalnej bazy danych w pliku, schematów, tabel i relacji (JOIN) może być obcy osobom spoza IT.

- **Konfiguracja Slack App (bot, scopes, OAuth & Permissions)** — tworzenie aplikacji Slack, dodawanie Bot Token Scopes, instalacja do workspace — to proces wymagający poruszania się po panelu deweloperskim Slacka.

## Prompt dla NotebookLM Deeper Search

Przygotowujesz materiał do podcastu wprowadzającego dla osób nietechnicznych (marketerzy, HR-owcy, pisarze, analitycy bez background'u IT), które uczą się korzystać z Claude Code i właśnie dotarły do tematu instalacji serwerów MCP.

Zbadaj i wyjaśnij przystępnym językiem następujące zagadnienia — tak, jakbyś tłumaczył je osobie, która nigdy nie pracowała w terminalu:

1. **Czym jest JSON i jak go edytować?** Wyjaśnij format na przykładzie przepisu kulinarnego (składniki = klucze, wartości = ilości). Jakie są najczęstsze błędy przy ręcznej edycji JSON i jak ich unikać?

2. **Node.js, npm i npx — po co to wszystko?** Dlaczego serwery MCP wymagają Node.js? Co robi npx i dlaczego jest wygodniejsze niż npm install? Użyj analogii: npx to jak "wypożyczenie narzędzia na chwilę" zamiast kupowania go na stałe.

3. **Zmienne środowiskowe — ukryte ustawienia systemu.** Czym jest zmienna środowiskowa? Dlaczego tokeny przechowujemy w zmiennych zamiast wpisywać w plik konfiguracyjny? Wyjaśnij na analogii sejfu z kodem: plik konfiguracyjny mówi "użyj kodu z sejfu", a sejf (zmienna środowiskowa) jest osobno i nie trafia do nikogo innego.

4. **Tokeny API i OAuth — "klucze do cudzego domu".** Czym token różni się od hasła? Dlaczego GitHub, Slack i Google wymagają tokenów? Co to jest OAuth i dlaczego przy Google Drive trzeba przejść przez "Google Cloud Console"? Wyjaśnij koncepcję "zakresów uprawnień" (scopes) — to jak oddanie klucza, który otwiera tylko jedno pomieszczenie, nie cały budynek.

5. **Ścieżki do plików — mapa Twojego komputera.** Czym jest ścieżka absolutna vs względna? Dlaczego `~` (tilde) nie działa wszędzie? Dlaczego Windows używa backslashy a Mac/Linux slashy? Pokaż to jako "adres pocztowy" — absolutny to pełny adres z miastem i ulicą, względny to "dwa domy dalej".

6. **Sandbox — ogrodzony plac zabaw.** Czym jest sandbox w kontekście bezpieczeństwa komputerowego? Dlaczego komendy Bash w Claude Code działają w sandboxie, ale serwery MCP już nie? Użyj analogii: Bash to gość, którego pilnujesz w domu. MCP to oddanie komuś kluczy — musisz mu ufać zanim je dasz.

7. **Git i tajemnice — dlaczego "usunięcie" pliku to za mało.** Czym jest git i historia commitów? Dlaczego wrzucenie tokenu do repozytorium jest niebezpieczne nawet jeśli potem go usuniesz? Co to jest .gitignore i jak chroni przed przypadkowym wyciekiem? Opowiedz historię z lekcji: developer commituje token, boty go znajdują w 15 minut.

8. **Prompt Injection — podstęp ukryty w danych.** Wyjaśnij jak atakujący może ukryć instrukcje w pliku, który Claude przeczyta i potraktuje jak polecenie użytkownika. Dlaczego AI nie odróżnia "danych" od "poleceń"? Użyj analogii: to jak podłożenie kartki z fałszywymi instrukcjami do skrzynki, z której asystent czyta Twoje polecenia.

Każdy z tych tematów omów w kontekście praktycznym — osoby słuchające podcastu za chwilę będą instalować swój pierwszy serwer MCP (filesystem), a potem ewentualnie GitHub, Slack czy SQLite.
