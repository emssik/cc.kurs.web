# Lekcja 02.02 — Custom Slash Commands: Twoja pierwsza automatyzacja (Research dla podcastu)

## Trudne tematy techniczne

- **Terminal / linia komend (mkdir, nano, ls, cd, mv, cp)** — lekcja zakłada swobodne poruszanie się po terminalu; osoby nietechniczne mogą nie wiedzieć, czym jest terminal ani jak uruchamiać komendy systemowe.

- **Struktura katalogów i ścieżki (~/.claude/commands/ vs .claude/commands/)** — koncept ścieżek bezwzględnych, katalogu domowego (~), ukrytych katalogów (z kropką na początku) i różnicy między lokalizacją osobistą a projektową wymaga zrozumienia systemu plików.

- **Git (git add, git commit, git push, git pull)** — udostępnianie komend zespołowi opiera się na systemie kontroli wersji; dla osób nietechnicznych git to czarna skrzynka.

- **Markdown (.md) i frontmatter (YAML między ---)** — lekcja wymaga tworzenia plików Markdown z opcjonalnym nagłówkiem YAML; oba formaty mogą być zupełnie obce.

- **Zmienne i placeholdery ($ARGUMENTS, $0, $1, $2)** — koncept zmiennych, które są dynamicznie zastępowane wartościami, to abstrakcja programistyczna nieoczywista dla laików.

- **Modele AI (haiku, sonnet, opus) i ich dobór** — rozróżnianie modeli pod kątem szybkości, kosztu i zdolności wymaga zrozumienia, czym w ogóle jest "model" i dlaczego jest ich kilka.

- **allowed-tools i kontrola uprawnień (Bash(git *), Read(*.csv))** — składnia glob patterns i koncept ograniczania narzędzi dostępnych dla AI to zaawansowany temat bezpieczeństwa.

- **Hot reload** — termin znany programistom (zmiana bez restartu), ale abstrakcyjny dla osób spoza branży.

- **Namespace, prefiksy, konflikty nazw** — organizacja komend przez prefiksy w nazwach plików i rozwiązywanie konfliktów nazw to koncept bliższy inżynierii oprogramowania.

- **Priority hierarchy (Enterprise > Project > Personal)** — hierarchia nadpisywania komend wymaga zrozumienia, jak systemy konfiguracji działają warstwowo.

- **CSV, EDA, CTR, SQL injection, XSS, N+1 queries** — lekcja rzuca wieloma skrótami technicznymi w przykładach, co może przytłoczyć osoby początkujące.

## Prompt dla NotebookLM Deeper Search

Przygotowuję podcast wprowadzający do lekcji kursu Claude Code o tworzeniu własnych komend (Custom Slash Commands). Słuchaczami są osoby nietechniczne: marketerzy, HR-owcy, pisarze, analitycy biznesowi — ludzie, którzy nigdy nie programowali i mogą nie znać terminala.

Lekcja pokazuje, jak tworzyć pliki .md z instrukcjami w naturalnym języku, które Claude Code zamienia w automatyczne komendy wywoływane przez /nazwa. Kluczowa bohaterka, Sara (marketerka), automatyzuje swój poranny raport kampanii.

Potrzebuję materiału, który w przystępny sposób wyjaśni następujące tematy:

1. **Czym jest terminal i jak się po nim poruszać** — co to jest mkdir, nano, ls, cd. Dlaczego w lekcji trzeba wpisywać komendy, a nie klikać myszką. Analogie do życia codziennego (np. terminal to rozmowa z komputerem tekstem zamiast kliknięciami).

2. **System plików i ścieżki** — co oznacza ~/ (katalog domowy), dlaczego niektóre foldery zaczynają się od kropki (.claude/) i są "ukryte", jaka jest różnica między ~/.claude/commands/ (osobiste) a .claude/commands/ (projektowe).

3. **Git dla zupełnie zielonych** — po co istnieje git, czym jest commit, push, pull. Dlaczego Sara musi "commitować" komendy, żeby Marek je dostał. Analogia: git to Google Docs dla kodu, ale z pełną historią zmian.

4. **Markdown i frontmatter** — czym jest plik .md, dlaczego to po prostu tekst z formatowaniem. Co to jest sekcja między --- na początku pliku (frontmatter/nagłówki) i po co służy (parametry komendy: opis, model, uprawnienia).

5. **Zmienne i placeholdery ($ARGUMENTS, $0, $1)** — dlaczego w tekście komendy pisze się $ARGUMENTS zamiast konkretnej nazwy pliku. Analogia: to jak zostawienie pustego miejsca w formularzu, które wypełniasz przy każdym użyciu.

6. **Modele AI — haiku, sonnet, opus** — dlaczego istnieją różne modele, czym się różnią (szybkość vs dokładność vs koszt). Kiedy wybrać który — proste porównanie do np. samochodu miejskiego vs SUV vs limuzyny.

7. **Uprawnienia narzędzi (allowed-tools)** — po co ograniczać to, co AI może robić. Czym jest glob pattern (np. *.csv). Dlaczego warto zabezpieczyć komendę, żeby nie zrobiła czegoś niechcianego.

Ton podcastu: przyjazny, bez żargonu, z dużą ilością analogii do codziennego życia. Cel: po wysłuchaniu słuchacz powinien rozumieć każdy element lekcji o Custom Slash Commands i czuć się gotowy do stworzenia swojej pierwszej komendy.
