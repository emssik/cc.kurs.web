# Lekcja 10 — Supermoce: jak pracuja eksperci (Research dla podcastu)

## Trudne tematy techniczne

- **Git checkout vs branch** — lekcja mowi o "osobnych kopiach repozytorium na dysku" vs branchach. Dla osoby nietechnicznej sam git jest obcy, a roznica miedzy checkout a branch to juz abstrakcja drugiego rzedu.

- **Rownolegle sesje w terminalu (taby, iTerm2, Windows Terminal)** — zaklada znajomosc pracy w terminalu, otwierania wielu instancji, zarzadzania tabami. Osoby nietechniczne czesto nie otworzyly terminala nigdy wczesniej.

- **Plan Mode / Auto-Accept Mode / Shift+Tab** — przelaczanie trybow pracy Claude za pomoca skrotow klawiszowych. Wymaga zrozumienia, ze Claude ma rozne "tryby" i ze prompt moze byc wykonany bez potwierdzania kazdej zmiany.

- **Subagenci (Task tool)** — koncept agenta uruchamiajacego inne agenty rownolegle. Wymaga zrozumienia delegowania zadan w kontekscie AI, overhead tokenow, rownoleglosci.

- **Extended Thinking i budzet tokenow** — co to jest "myslenie" modelu, czym sa tokeny, co oznacza 31 999 vs 63 999 tokenow budzetu. Dla nietechnicznych — caly koncept tokenow jest nowy.

- **Slash commands i pliki .md w katalogu .claude/commands/** — tworzenie wlasnych komend poprzez pliki markdown w ukrytym katalogu. Zaklada znajomosc struktury katalogow, plikow konfiguracyjnych i konwencji dotowych katalogow.

- **Conventional Commits i PR (Pull Request)** — lekcja uzywa terminow z workflow programistycznego (commit, branch, push, PR, diff). Dla nietechnicznych to obcy jezyk.

- **MCP (Model Context Protocol) i instalacja narzedzi CLI** — VoiceMode MCP instalowany przez `claude mcp add`. Wymaga zrozumienia co to MCP, co to CLI, co to scope.

- **Vibe coding i pipeline transkrypcji (Groq/Whisper)** — lancuch narzedzi: mowa -> transkrypcja -> korekta -> formatowanie. Zaklada znajomosc Whisper, Groq, pipeline'ow przetwarzania.

- **One-shot execution i /rewind** — koncepcja wykonania zadania "w jednym strzale" i cofania sie do poprzedniego stanu. Bliska gitowemu revert/reset, ale w kontekscie sesji Claude.

- **CLAUDE.md jako plik konfiguracyjny** — plik sterujacy zachowaniem agenta. Zaklada zrozumienie, ze AI mozna "konfigurowac" plikami tekstowymi w repozytorium.

## Prompt dla NotebookLM Deeper Search

Przygotowuje podcast dla osob poczatkujacych (marketerow, HR-owcow, pisarzy, analitykow) o zaawansowanych technikach pracy z Claude Code. Potrzebuje przystepnego wprowadzenia do nastepujacych tematow, ktore pojawiaja sie w lekcji "Supermoce — jak pracuja eksperci":

1. **Co to jest git i po co kopiowac repozytorium?** Wyjasnienie czym jest repozytorium kodu, czym sie rozni checkout (kopia katalogu) od brancha, i dlaczego eksperci pracuja na kilku kopiach jednoczesnie — uzyj analogii do fizycznych folderow z dokumentami na biurku.

2. **Tryby pracy Claude Code: Plan Mode i Auto-Accept.** Wyjasnienie ze Claude moze najpierw planowac (nie ruszajac plikow), a potem wykonywac plan automatycznie. Porownaj to do rozmowy z asystentem: najpierw omawiasz plan, potem mowisz "dzialaj". Wspomnij o /rewind jako "cofnij" i Shift+Tab jako przelaczaniu trybow.

3. **Co to sa subagenci AI?** Wyjasnienie koncepcji jednego agenta delegujacego zadania do kilku pomniejszych agentow dzialajacych rownolegle. Uzyj analogii do managera, ktory rozdziela zadania w zespole. Wspomnij o koszcie (tokeny) i o tym, ze mozna zachecic Claude do czestszego uzycia subagentow przez plik konfiguracyjny.

4. **Tokeny i "myslenie" modelu AI.** Wyjasnienie czym sa tokeny (jednostki tekstu), dlaczego model "mysli" przed odpowiedzia (Extended Thinking), co oznacza budzet tokenow na myslenie, i jak uzytkownik moze podejrzec ten proces (Ctrl+O).

5. **Slash commands — wlasne komendy uzytkownika.** Wyjasnienie ze mozna tworzyc gotowe "przepisy" dla Claude w plikach tekstowych, ktore potem wywoluje sie jednym poleceniem. Pokaz przyklady nietechniczne: generowanie raportu tygodniowego, ogloszenia o prace, content briefu.

6. **MCP i narzedzia glosowe.** Wyjasnienie czym jest MCP (Model Context Protocol) w jednym zdaniu, a nastepnie skupienie sie na praktyce: jak dyktowac polecenia glosem zamiast pisac, jakie narzedzia sa dostepne (SuperWhisper, Wispr Flow, VoiceMode), i dlaczego mowienie jest 3-5x szybsze od pisania.

Ton podcastu: przyjazny, bez zadnego wyzszosciowego tlumaczenia. Sluchacz to inteligentna osoba, ktora po prostu nie miala stycznosci z terminalem i programowaniem. Uzyj duzo analogii z codziennego zycia biurowego. Kazdy temat powinien byc wyjasniany tak, jakbys tlumaczyc go koledze przy kawie. Jezyk: polski.
