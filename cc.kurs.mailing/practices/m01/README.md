# TechStart: Mini-Kurs Praktyczny

Witaj w praktycznej czesci Modulu 1 kursu Claude Code!

## Co to jest?

To mini-kurs, w ktorym zastosujesz wszystko, czego nauczyles sie w Module 1. Zamiast kolejnej teorii - realne zadania w realistycznym scenariuszu.

**Scenariusz:** Jestes nowym koordynatorem w organizacji TechStart. Za 2 tygodnie Dzien Otwarty, poprzednik zostawil chaos, a Ty masz tylko Claude Code. Wystarczy?

Spoiler: tak, wystarczy.

## Wymagania wstepne

Przed rozpoczeciem upewnij sie, ze:

- Ukonczyles **Modul 1** kursu Claude Code (lekcje 00-09)
- Masz zainstalowanego i skonfigurowanego **Claude Code** (CLI)
- Masz okolo **1.5-2 godziny** na intensywna prace

## Jak zaczac?

### Krok 1: Przejdz do folderu

```bash
cd /sciezka/do/practices/m01
```

### Krok 2: Uruchom Claude Code

```bash
claude
```

### Krok 3: Rozpocznij pierwsza lekcje

```
/start-p-1
```

To wszystko! Claude przeprowadzi Cie przez caly mini-kurs.

## Struktura mini-kursu

| Lekcja | Temat | Co robisz | Czas |
|--------|-------|-----------|------|
| **P.1** | Analiza | Przegladasz odziedziczony chaos, rozumiesz co masz | ~20 min |
| **P.2** | Planowanie | Tworzysz plan i CLAUDE.md dla projektu | ~25 min |
| **P.3** | Tworzenie | Piszesz materialy komunikacyjne z Claude | ~35 min |
| **P.4** | Review | Konsultujesz wyniki z "zespolem" (agentami) | ~25 min |

**Laczny czas:** ~1.5-2 godziny

## Czego sie nauczysz?

### Umiejetnosci praktyczne

- Szybka analiza nieurzadkowanej dokumentacji
- Tworzenie efektywnego CLAUDE.md dla projektu
- Generowanie tresci z zachowaniem tonu marki
- Iteracyjne poprawianie wynikow na podstawie feedbacku

### Techniki Claude Code

- Praca z kontekstem projektowym
- Wykorzystanie slash commands
- Konsultacje z agentami
- Zarzadzanie outputem

## Co otrzymasz na koniec?

Po ukonczeniu mini-kursu bedziesz miec:

1. **Analize sytuacji** - porzadek w odziedziczonym chaosie
2. **CLAUDE.md projektu** - dokumentacje dla przyszlej pracy
3. **Materialy komunikacyjne** - gotowe do uzycia (email, brief, etc.)
4. **Doswiadczenie** - praktyczna wiedza, ktora zastosujesz w swoich projektach

## Warianty zadan

Mini-kurs oferuje 2 warianty - wybierz odpowiedni dla siebie:

- **Wariant A (programistyczny):** Wiecej automatyzacji, praca z danymi CSV, skrypty
- **Wariant B (biurowy):** Fokus na tresciach, dokumentach, organizacji informacji

Mozesz wybrac jeden wariant lub mieszac zadania z obu.

## Potrzebujesz pomocy?

W trakcie mini-kursu:

```
/pomoc
```

Claude podpowie, co robic dalej.

## Najczesciej zadawane pytania

### Czy moge zrobic przerwe w trakcie?

Tak! Stan Twojej pracy jest zapisywany w folderze `output/`. Mozesz wrocic pozniej i kontynuowac.

### Co jesli zrobie cos zle?

Nic sie nie stanie. To jest nauka - eksperymentuj swobodnie. Jesli cos poszlo nie tak, zawsze mozesz zaczac dana lekcje od nowa.

### Czy musze robic wszystkie lekcje po kolei?

Zalecamy tak - kazda lekcja buduje na poprzedniej. Ale jesli juz masz doswiadczenie, mozesz przeskoczyc do interesujacych Cie czesci.

### Ile razy moge powtarzac mini-kurs?

Ile chcesz! Kazde podejscie moze dac inne wyniki - to zaleta pracy z AI.
