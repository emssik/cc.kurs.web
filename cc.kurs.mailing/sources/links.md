## Kluczowe wzorce z materiałów:

### Workflow Patterns
- **Phased Development**: Research → Plan → Implement → Validate (clear context between phases)
- **60% Context Rule**: Never exceed 60% context usage
- **Plan Mode First**: Use keywords like "analyze", "think" before coding
- **Parallel Subagents**: Different agents researching in parallel

### CLAUDE.md Best Practices
- Nazywane "constitution" projektu
- Automatycznie wczytywane na początku każdej sesji
- Zapisywanie powtarzających się błędów i wzorców

### Mental Models
- "Post-Junior developer with experience but poor memory"
- "Fast intern with perfect memory"
- "Junior developer who doesn't learn" (wymaga ciągłego nadzoru)

### Cost Considerations
- Senior engineer going all-in: $1000-1500/month
- "Thinking depth" keywords: think < think hard < think harder < ultra think

### Key Success Factors
- Clear specs and expectations
- Interactive work (one change at a time)
- Experienced developers who know good output
- Treating as onboarded engineer with proper documentation

### Common Pitfalls
- Over-engineering when specs are vague
- First attempt often "95% garbage" (requires iteration)
- Better at focused tasks than doing "too much" at once
