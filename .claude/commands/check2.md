# Wytyczne stylistyczne dla lekcji Claude Code

## Twoje zadanie

Jeśli podano plik: **$ARGUMENTS** — przeczytaj go i sprawdź pod kątem poniższych wytycznych.

Jeśli nie podano pliku — potraktuj to jako materiał referencyjny przy pisaniu/edytowaniu lekcji.

**Zasada nadrzędna:** poprawiaj styl i klarowność bez zmiany sensu.  
Jeśli widzisz problem merytoryczny, zgłoś go osobno jako „Sugestia merytoryczna” (bez przerabiania treści).

---

## Wytyczne stylistyczne

### 1. Unikaj przesady
AI uwielbia przesadę. Wszystko jest „kluczowe”, a każdy szczegół „podkreśla” coś głębszego. Ludzie tak nie mówią.

**Wyrzuć lub osłab:**
- „kluczowe”, „fundamentalne”, „niezwykle istotne”
- „znacząco”, „drastycznie”, „diametralnie” (jeśli nie ma liczb/dowodu)
- każde słowo sugerujące, że coś jest ważniejsze niż naprawdę jest

---

### 2. Wyrzuć puste słowa
Wyrzuć „rewolucję” i „game-changery”. To puste, nadużywane słowa.

**Unikaj:**
- rewolucja, rewolucyjny
- game-changer
- przełomowy
- transformacyjny

---

### 3. Unikaj sztucznych kontrastów
Konstrukcje typu „To nie jest X. To Y” brzmią jak tania dramaturgia, a nie argument.

**❌ Źle:**
> „To nie jest narzędzie. To partner w pracy.”

**✅ Lepiej:**
> „Claude Code pomaga Ci w codziennej pracy.”

---

### 4. Napraw tempo — nie bądź metronomem
AI pisze zdania o tej samej długości. Zmieniaj rytm, by brzmieć naturalnie.

**❌ Źle:**
> „Claude Code jest szybki. Claude Code jest precyzyjny. Claude Code jest pomocny.”

**✅ Lepiej:**
> „Claude Code jest szybki i precyzyjny. A co najważniejsze — naprawdę pomaga.”

---

### 5. Skasuj meta-komentarze
Nie ogłaszaj: „W tej sekcji…”, „Podsumowując…”, „Teraz przejdźmy do…”.

**Usuń:**
- „W tej sekcji omówimy…”
- „Podsumowując…”
- „Teraz przejdźmy do…”
- „Jak widzieliśmy powyżej…”
- „Omówmy teraz konsekwencje…”

Po prostu pisz treść — nie zapowiadaj jej.

---

### 6. Pisz tak, jak mówisz
Używaj „ja” i „ty”. Unikaj strony biernej, którą AI faworyzuje.

**❌ Źle:**
> „Zostanie wyświetlony komunikat o błędzie.”

**✅ Lepiej:**
> „Zobaczysz komunikat o błędzie.”

---

### 7. Usuń zbędne podsumowania
Nie streszczaj czytelnikowi tego, co przeczytał dwa akapity wyżej.

**Usuń sekcje typu:**
- „Podsumowując powyższe…”
- „Jak omówiliśmy…”
- „Wracając do wcześniejszego punktu…”

---

### 8. Formatuj z umiarem
Tekst nie może wyglądać jak korporacyjny szablon, w którym każde zdanie jest nagłówkiem.

**❌ Źle:**

Wprowadzenie

Cel

Kontekst

Założenia

Szczegóły

Punkt 1

Podpunkt 1a

**✅ Lepiej:**
Używaj nagłówków, gdy naprawdę zmieniasz temat. Nie każdy akapit potrzebuje własnego nagłówka.

---

### 9. Zajmij stanowisko
Zamiast asekuracyjnego „to podejście może potencjalnie…”, napisz wprost, co działa.

**❌ Źle:**
> „To rozwiązanie może potencjalnie przynieść pewne korzyści w niektórych sytuacjach.”

**✅ Lepiej:**
> „To rozwiązanie działa.”

**Uwaga:** jeśli to nie jest fakt, tylko doświadczenie/heurystyka, zaznacz to krótko (np. „u mnie działa”, „zwykle działa”).

---

### 10. Zamień „zasadę trzech”
Triady typu „szybkość, wydajność i innowacja” są wyświechtane.

**❌ Unikaj:**
- „szybkość, wydajność i innowacja”
- „precyzja, jakość i niezawodność”
- „elastyczność, skalowalność i bezpieczeństwo”

Jeśli wymieniasz kilka rzeczy — każda ma wnosić coś konkretnego.

---

### 11. Bądź konkretny
Ogólniki brzmią jak robot generujący tekst „na akord”.

**❌ Źle:**
> „Claude Code znacząco usprawnia workflow developerski.”

**✅ Lepiej:**
> „Z Claude Code napisałem ten endpoint w 15 minut zamiast godziny.”

---

### 12. Tnij watę i oczywistości
AI dopisuje oczywiste zdania i „miękkie wprowadzenia”.

**Usuń lub skróć:**
- definicje rzeczy oczywistych dla odbiorcy
- „warto pamiętać, że…”, „należy zauważyć, że…”, „jak wiemy…”
- zdania, które nic nie zmieniają (brak decyzji, brak kroku, brak informacji)

---

### 13. Unikaj „to zależy” bez warunków
Jeśli piszesz „to zależy”, dopisz warunki.

**❌ Źle:** „To zależy od sytuacji.”  
**✅ Lepiej:** „Jeśli masz X — zrób A. Jeśli Y — zrób B.”

---

### 14. Preferuj czynności i następny krok
Każdy większy fragment powinien kończyć się: decyzją / konkretnym krokiem / mini-zadaniem / pytaniem kontrolnym.

**Źle:** opis bez konsekwencji.  
**Lepiej:** „Zrób teraz X i sprawdź Y.”

---

### 15. Nie rozwadniaj pewności
Nie brzmi naturalnie, gdy każde zdanie jest „może/prawdopodobnie/potencjalnie”.

**Unikaj kumulacji:**  
„może”, „potencjalnie”, „w pewnym sensie”, „raczej”, „wydaje się” — jeśli nie są potrzebne.

---

## Szybka checklista

Przed publikacją lekcji sprawdź:

- [ ] Czy usunąłem słowa: kluczowe, fundamentalne, rewolucyjne, game-changer, przełomowe?
- [ ] Czy unikam konstrukcji „To nie jest X. To Y”?
- [ ] Czy zdania mają różną długość?
- [ ] Czy usunąłem meta-komentarze typu „Podsumowując”?
- [ ] Czy używam „ty/ja” zamiast strony biernej?
- [ ] Czy mam konkretne przykłady zamiast ogólników?
- [ ] Czy zajmuję stanowisko i nie uciekam w „potencjalnie”?
- [ ] Czy formatowanie jest umiarkowane?
- [ ] Czy uciąłem watę i oczywistości?
- [ ] Czy „to zależy” ma warunki?
- [ ] Czy po większej sekcji jest jasny następny krok?

## Rozpocznij sprawdzanie

Jeśli podano plik **$ARGUMENTS**, przeczytaj go teraz i:

1. Znajdź fragmenty łamiące powyższe wytyczne.
2. Dla każdego fragmentu podaj:
   - Numer linii
   - Cytat problematycznego tekstu
   - Która zasada została złamana
   - Poziom problemu (S1/S2/S3)
   - Propozycja poprawki w formie „Przed → Po” (bez zmiany sensu)
3. Na końcu dodaj:
   - „Powtarzające się tiki” (najczęstsze typy problemów)
   - Maksymalnie 5 sugestii merytorycznych (jeśli są), oddzielnie od stylistyki

### Skala problemów
- **S1:** mocno brzmi jak AI / psuje naturalność / rozwadnia przekaz
- **S2:** średnio przeszkadza / da się czytać, ale warto poprawić
- **S3:** kosmetyka / drobna poprawa rytmu lub słownictwa

### Limity (żeby raport był użyteczny)
- Jeśli problemów jest dużo: wypisz **maks. 25** najważniejszych (priorytet S1, potem S2).
- Resztę streść w „Powtarzających się tikach”.
